Découverte du Java

Nous allons maintenant écrire notre tout premier programme Java. Dans un programme informatique (en Java ou autre), on trouve deux choses : les données, et les instructions qui manipulent ces données.

Les données en Java

Java permet d'utiliser différents types de données, tels que les entiers, les nombres à virgules ou les chaînes de caractères. Si on veut utiliser une donnée plusieurs fois, il faut la stocker dans une variable, qui est une sorte de case contenant une valeur. Java est un langage typé, ce qui veut dire que l'on ne peut stocker une valeur que dans une variable du bon type. Pas question de stocker une chaîne de caractères dans une variable entière.

Pour déclarer (i.e. créer) une variable, il suffit d'écrire son type, un espace et le nom de la variable. Parmi les types existants, on trouve: int pour les entiers, double pour les nombres à virgule, boolean pour les booléens (i.e. les variables qui peuvent être soit vraies soit fausses) et String pour les chaînes de caractères. Si l'on veut, on peut faire suivre la déclaration du signe = suivi de la valeur initiale à donner à la valeur.

Ainsi, pour créer une variable nommée x contenant des entiers, on écrira : 

int x;
Si on veut que sa valeur initiale soit 5, on écrira : 
int x = 5;

Plus tard dans le programme, si l'on souhaite affecter une nouvelle valeur à la variable, c'est très simple : 

x=3; ceci remplace la valeur de x par 3

Généralités sur les instructions

Le programme Java le plus simple est formé d'une suite d'instructions, séparées par des points-virgules. Il faut le comprendre comme une suite d'ordres simples que l'on donne à la machine. C'est assez comparable à une recette de cuisine où l'on dit cassez les oeufs puis ajoutez du sel puis mélangez le tout puis faites cuire.

Les affectations de variables comme 

x=5
sont les instructions les plus simples. L'autre type classique d'instruction en Java est l'appel à une méthode. Nous verrons plus tard comment définir de telles méthodes, disons simplement pour l'instant qu'il s'agit des ordres disponibles. La syntaxe est en général est : 
nomDeLaMethode()

L'exemple ci-dessus de recette s'écrirait donc ainsi à peu près ainsi: 

casserLesOeufs();
ajouterDuSel();
melangerLeTout();
faireCuire();

Les instructions en Java

 Bien entendu, ces méthodes n'existent pas en Java, et il est difficile de faire la liste des méthodes existantes en Java. Dans cette leçon, les instructions les plus souvent utilisées sont des appels à des méthodes de la buggle, par lesquelles on donne des ordres à la buggle qui les reçoit.

Par exemple, il y a une méthode pour chaque bouton du contrôle interactif. Pour faire la même chose que le bouton forward (faire avancer la buggle d'un pas), il faut écrire dans l'éditeur : 

forward();
De même, pour faire l'équivalent des boutons backward, turn left et turn right, il faut utiliser respectivement : 
backward();
turnLeft();
turnRight();
Le bouton mark est un peu particulier, puisqu'il correspond à deux méthodes : la première lève le crayon, tandis que la seconde le baisse. 
brushDown();
brushUp();

La buggle offre d'autres méthodes, que nous verrons lorsque le besoin s'en fera sentir.

Ajouter des commentaires

Quand on écrit un programme un peu complexe, il est parfois utile d'ajouter des commentaires pour simplifier la relecture du code après coup. Il existe trois types de commentaires en Java:
  • Commentaires sur une seule ligne. Dès que le compilateur rencontre le symbole //, il ignore la fin de la ligne.
  • Commentaires sur plusieurs lignes. Le compilateur ignore tout ce qui se trouve entre les symboles /* et */, même s'ils sont sur des lignes différentes
appelMethodeLuParLeCompilateur();  // tout ceci est ignoré
autreAppel(); /* ceci est
               également ignoré */
encoreUnAppel();

Il existe une troisième forme de commentaires, entre /** et */, qui sont lus par un programme nommé JavaDoc pour générer automatiquement la documentation expliquant comment utiliser le code. Ces commentaires doivent suivre un formalisme précis.

Objectif de cet exercice

 Nous allons faire le programme Java pour buggle le plus simple existant : vous devez faire en sorte que la buggle avance d'un pas.

Pour cela, écrivez simplement dans l'éditeur (onglet Source Code) le code suivant : 

forward();
N'oubliez pas le point-virgule, qui indique au compilateur que l'instruction est finie.

Après avoir recopié le code dans l'éditeur, cliquez sur run et passez à l'exercice suivant.
