Maintenant que vous savez écrire des choses sur le sol, nous allons utiliser cette compétence pour dessiner un merveilleux G par terre. Consultez le panneau d'objectif pour les détails de ce qui est attendu.

When you write a quite complex program, it is sometimes useful to add comments to simplify the code reviews afterward. Here for example, it's quite easy to get lost in the drawing process, and you may want to add comments like vertical bar done or finished drawing the G. Time to move back to initial position. Commenting your code is almost mandatory if you (or someone else) want to read it afterward, although overcommenting (describing obvious stuff) is a bad idea as the important idea get lost in the noise. There is threetwo types of comments in Java, instructing the compiler to not read the text you add for humans:

  • Comments on a single line. When the compiler encounters the symbols //, it ignores the end of the line.
  • Comments on a single line. When the compiler encounters the symbol #, it ignores the end of the line.
  • Comments on several lines. The compiler ignores anything placed between the symbols /* and */ even if they are placed on differing lines.
  • Comments on several lines. The compiler ignores anything placed between a line beginning with ''' and the next line ending with '''.
methodCallReadByTheCompiler(); // all this is ignored
otherCall(); /* This is
               also ignored */
yetAnotherCall();

methodCallReadByTheCompiler() # all this is ignored
otherCall() 
''' This is
also ignored  '''
yetAnotherCall();

Il existe une troisième forme de commentaires, entre /** et */, qui sont lus par un programme nommé JavaDoc pour générer automatiquement la documentation expliquant comment utiliser le code. Ces commentaires doivent suivre un formalisme précis.

The comments on several lines are often used to document functions and classes.
